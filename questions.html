  7. Prompt: Below is a callback. Explain what that is and what are good uses for this JS pattern. 

                                function sayTime(time) {
                              alert('The time is ' + time);
                            }

                            function findTime(callback) {
                              time = new Date()
                              callback(time);
                            }
    
                            findTime(sayTime);

    Answer:

    A callback is basically a function that is passed as an argument onto another function. The purpose here is to decouple each action (separation of concern): the only goal of findTime is to fetch the current time, then pass it onto its callback. And the only goal of sayTime is to say what time it is. Hence we can alter the behavior of one function without touching another.

    Callbacks are used extensively in Javascript, and have many applications, such as:

    • Mapping: applying a function onto each element of an array
    • Triggering an action after fetching an HTTP request
    • Handling a browser's event (click on a button, for example)

  10. Prompt: Please explain - to a non-technical person - the difference between server side and client side rendering. Be sure to include how templates and AJAX relate to the different ways to render content.

    Answer:

    Server-side rendering means that the content is generated by the server application itself, i.e. the server application includes the HTML content (to be displayed) directly in the HTTP response, which will be returned to the browser and displayed as is. The rendered content can of course be dynamic. In that case, the server application manages the changing content via HTML templates, which is basically an HTML file, including some variables, changed on the fly by the application (for example, all product pages are derived from the same HTML file). This is historically by far the most common way of delivering content on the web.

    Client-side rendering is a bit more tricky: in this case, the Javascript code on the client side (which has been returned from the server to the browser, through an HTTP response) has the responsability of displaying the HTML content properly. Typically, the Javascript code performs a series of HTTP requests (using AJAX for example) to different services (located on the same server, or on several servers), to fetch the adequate content. Once fetched, the Javascript code arrange and display the content for the user. This method of rendering has gain a lot of popularity in the last few years, through the adoption of Javascript frameworks such as React, Vue or Angular JS.
